# Growth — Étude poussée : funnel d’acquisition et KPIs

**Rôle** : Cadre pour l’agent **Growth Hacker / OSINT** lorsqu’il réalise une **étude poussée** sur un prospect : **KPIs** pertinents et **pistes d’amélioration du funnel d’acquisition**.

**Pilote** : Growth (règle `.cursor/rules/growth.mdc` — ingénierie KPI + marketing digital ; `expert-google-dorks-linkedin.mdc`, `pentester.mdc` en spécialisations).  
**Collaboration** : Chef de Projet (validation), Expert SEO (cohérence avec rapport lead magnet), Data Analyst (scoring, analytics).

---

## 1. Objectif de l’étude poussée

- **KPIs** : proposer les indicateurs pertinents à suivre pour ce prospect (acquisition, trafic, qualification, conversion, coût par lead, etc.).
- **Funnel d’acquisition** : identifier les étapes du funnel (awareness → considération → conversion → rétention si pertinent), les points de friction et les **pistes d’amélioration** (canaux, messages, CTA, qualification, automatisation).

L’étude complète le **rapport SEO** (Expert SEO) et alimente la stratégie de la **landing** et du lead magnet.

---

## 2. Livrable type

- Note ou rapport dans le **dossier contact** : `docs/contacts/<slug_contact>/etude-growth-funnel-kpis.md` (ou nom équivalent).
- Contenu suggéré :
  - KPIs recommandés (liste + justification courte).
  - Schéma ou description du funnel actuel / cible.
  - Pistes d’amélioration (priorisées si possible).
  - Synergies avec le rapport SEO et le wording/copywriting (Expert SEO).

---

## 3. Quand la demander

- À la demande du **Conseiller** ou du **Chef de Projet** pour un prospect donné (ex. premier rapport SEO + première landing).
- Référence segmentation : `segmentations/2025-01-30-premier-rapport-seo-landing-p4s-archi.md` (exemple P4S-archi).
